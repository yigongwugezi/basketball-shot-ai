# 3D 骨架与角度误差路线

生成日期：2026-06-18

## 核心判断

当前 MVP 里计算的肘角、膝角、躯干角，必须定义为“2D 画面投影角”，不能直接宣传成真实生物力学关节角。

原因很简单：同一个真实手肘动作，从侧面、45 度、正面、后方拍摄，投影到画面上的夹角会明显变化。拍摄角度越偏，2D 角度和真实 3D 角度的误差越大。

所以本项目的专业化路线不能停留在 2D 骨架，而要为 3D 姿态、相机标定和多机位重建预留接口。

## 阶段 1：MVP 处理方式

目标：先做可用，不假装专业。

- 只支持单人、固定机位、全身入镜。
- 优先支持侧面和 45 度机位。
- 报告里把角度标注为“画面角度”或“2D 估计角度”。
- 对明显不合适的机位给出质量提醒。
- 同一用户多次训练时，优先比较同一拍摄角度下的趋势，而不是绝对角度。

MVP 可以做的指标：

- 出手帧肘部是否充分伸展。
- 下沉阶段膝盖弯曲是否明显。
- 出手时躯干是否大幅前倾或后仰。
- 随球动作是否完整。
- 多次投篮之间的动作一致性。

MVP 不应该做的承诺：

- 不承诺真实 3D 肘角。
- 不承诺真实 3D 膝角。
- 不承诺所有拍摄角度都同样准确。
- 不用单目 2D 结果下绝对的专业诊断结论。

## 阶段 2：拍摄角度识别与置信度

目标：让系统知道“这个视频适不适合分析”。

需要新增：

- camera_view：side、front、back、45_degree、unknown。
- view_confidence：机位判断置信度。
- metric_confidence：每个指标的可信度。
- warning：不适合的拍摄角度提示。

可用方法：

- 用人体左右肩、左右髋、双脚朝向判断机位。
- 用篮筐、地面线、球员朝向辅助判断。
- 对不同机位使用不同规则，不再所有视频套同一套角度阈值。

## 阶段 3：单目 3D 骨架

目标：用一个普通视频估计 3D 骨架，降低拍摄角度影响。

可选技术路线：

- 2D pose detector：RTMPose、RTMW、YOLO pose、ViTPose。
- 2D-to-3D lifting：VideoPose3D、MotionBERT、MHFormer、PoseFormer 系列。
- 输出 3D skeleton 后，再计算 3D 肘角、膝角、躯干角。

优点：

- 用户只需要一个手机视频。
- 产品门槛低，适合大众使用。
- 可以作为专业版之前的增强功能。

局限：

- 单目 3D 本质上仍有深度歧义。
- 对投篮这种手臂遮挡、球遮挡、出手瞬间高速运动的场景，误差仍然存在。
- 不能把单目 3D 结果直接当作严格生物力学测量。

产品结论：

单目 3D 值得做，但适合做“更稳的姿态估计”和“专业感增强”，不适合作为最终专业精度的唯一基础。

## 阶段 4：双机位或多机位 3D 专业模式

目标：真正降低拍摄角度导致的关节角误差。

推荐路线：

- 两台手机，分别拍摄侧面和正面/45 度。
- 做相机标定或简化标定。
- 对同一时刻的 2D 关键点做三角化。
- 重建 3D 人体骨架和 3D 球轨迹。
- 在 3D 空间中计算真实关节角、出手角、入框角。

现实可参考方向：

- OpenCap：面向运动与生物力学的无标记动作捕捉系统，使用多视角视频估计人体运动。
- Pose2Sim：基于多视角视频的开源无标记运动学分析流程。

产品结论：

双机位/多机位是未来专业版最靠谱的方向。它比单目 3D 麻烦，但更接近真实运动测量，也更适合打“专业训练工具”的定位。

## 工程接口预留

从现在开始，后端数据结构应预留这些字段：

```json
{
  "camera": {
    "view": "side",
    "view_confidence": 0.82,
    "calibrated": false
  },
  "pose_2d": [],
  "pose_3d": null,
  "metrics": {
    "elbow_angle_2d": null,
    "elbow_angle_3d": null,
    "metric_confidence": null
  },
  "warnings": [
    "当前角度为 2D 画面投影角，受拍摄机位影响。"
  ]
}
```

## 版本安排

V1：单机位 2D 骨架 MVP。

V2：机位识别、质量门槛、指标置信度。

V3：单目 3D 骨架实验版。

V4：双机位采集流程和同步流程。

V5：多机位 3D 骨架、3D 球轨迹、专业报告。

## 当前决策

现在先不做 3D，但代码和产品文案必须承认 2D 角度限制。

当前 MVP 的重点仍然是：

- 视频上传流程跑通。
- 关键帧展示清楚。
- 骨架检测稳定。
- 出手帧识别逐步准确。
- 报告能让用户看懂“哪里可能有问题”。

等 2D 流程稳定后，再进入单目 3D 骨架实验。
