# 项目想法制作版总结

生成日期：2026-06-15

这份文档只保留和“真正做项目”有关的内容：项目定位、目标用户、核心功能、MVP、差异化、风险、验证方式和优先级。它不是聊天记录归档，也不是情绪总结。

## 1. 当前所有项目方向

目前保留的项目方向有 5 个：

1. 老人微信防骗/截图陪跑/远程照护助手。
2. AI 运动动作纠正教练。
3. 微信聊天记录/截图转纪念册、文档、证据包。
4. AI C 盘解释器/电脑空间清理/桌面整理助手。
5. 可疑内容/AI 深度伪造风险判断助手。

另外有一个方法论方向：

- 到评论区、差评区、求助帖里“狙击需求”，收集真实用户说的“要是有……就好了”。

## 2. 项目一：老人微信防骗与截图陪跑助手

### 2.1 一句话定位

给子女买单的父母防骗与远程照护助手。老人通过微信发截图、语音和可疑内容求助，AI 先做防骗判断和手机操作陪跑，遇到高风险时通知子女兜底。

### 2.2 原始想法

- 老年人使用电子产品困难。
- 老人最熟悉的入口大概率是微信。
- 希望把 AI 做进微信里，让老人扫码添加，像和好友聊天一样使用。
- 模型可以用豆包、通义、Kimi、DeepSeek 等，不局限于腾讯元宝。
- 不要做一个老人版豆包，而是做家庭关系里的 AI 助手。

### 2.3 目标用户与付费方

使用者：

- 老人。

主要付费者：

- 子女。

潜在 B 端付费方：

- 养老机构。
- 陪诊/护理/家政服务商。
- 医院。
- 药店。
- 保险公司。
- 社区/街道。
- 政府采购。
- 智能硬件厂商。

### 2.4 老人端产品形态

理想形态：

- 像微信好友一样存在。
- 老人扫码添加。
- 支持文字、语音、图片、截图。
- 老人不用下载新 App。

现实路径：

- 微信原生 AI 好友能力未必开放给第三方。
- 个人微信机器人风险高，不适合长期产品化。
- 更现实的是企业微信客户联系、微信客服、公众号/服务号等类好友形态。

二维码逻辑：

- 子女创建父母档案。
- 系统生成专属二维码。
- 老人扫码添加。
- 后台把老人微信身份与家庭档案绑定。
- 不是每个家庭一个独立机器人，而是统一 AI 助手 + 每个家庭独立配置。

### 2.5 子女端功能

子女端是这个产品的商业核心。

功能：

- 创建父母档案。
- 填写城市、常去医院、常用药、紧急联系人、常见风险。
- 设置风险提醒规则。
- 设置用药、复诊、缴费、天气等提醒。
- 查看重要聊天摘要。
- 接收高风险通知。
- 必要时远程介入。

### 2.6 核心功能

#### 2.6.1 防诈骗判断

老人可以把这些内容发给助手：

- 陌生短信。
- 医保退款。
- 快递异常。
- 中奖信息。
- 投资理财。
- 贷款。
- 收款码。
- 验证码。
- 陌生链接。
- 评论区引导加微信。
- 直播购物诱导。

AI 输出要非常明确：

- 先不要点。
- 不要输入验证码。
- 不要转钱。
- 先问家里人。

高风险时通知子女。

边界：

- 不能监控微信钱包。
- 不能监控银行卡流水。
- 不能承诺“任何转账超过 200 元提醒我”。
- 只能基于老人主动发给助手的内容或对话里的风险意图判断。

#### 2.6.2 截图陪跑

老人不会操作手机时，发截图给助手。

AI 不长篇解释，而是一步一步告诉：

- 先点哪里。
- 不要点哪里。
- 点完再截图发我。
- 如果出现验证码、银行卡密码、付款页面，先停下。

适用场景：

- 挂号。
- 缴费。
- 找小程序。
- 设置手机。
- 看通知。
- 微信/支付宝/快手/抖音/淘宝/拼多多基础流程。

#### 2.6.3 高风险通知子女

触发场景：

- 验证码。
- 转账/付款。
- 收款码。
- 陌生链接。
- 退款客服。
- 投资理财。
- 贷款。
- 保健品。
- 免费体检。
- 低价旅游。

通知形式：

- 给子女端生成摘要。
- 告诉子女老人遇到了什么、AI 已经如何提醒。

#### 2.6.4 子女远程介入

高风险或 AI 无法解决时，子女可以接管。

示例：

- 老人问：“他说让我把验证码发过去。”
- AI 提醒老人暂停。
- 子女收到通知。
- 子女进入对话或给出指示。

#### 2.6.5 用药、复诊和生活提醒

功能：

- 吃药提醒。
- 复诊提醒。
- 量血压/血糖提醒。
- 取药/买药提醒。
- 水电燃气缴费提醒。
- 天气穿衣/带伞提醒。
- 节日、生日问候提醒。

边界：

- 只做提醒和记录。
- 不做医疗诊断。
- 不做药物剂量建议。
- 不做漏服处理、禁忌分析等高风险医疗判断。

#### 2.6.6 家庭知识库

子女可配置：

- 父母所在城市。
- 常去医院。
- 常用药。
- 常用药店。
- 物业电话。
- 宽带客服电话。
- 常用打车地址。
- 常用小程序。
- 紧急联系人。
- 老人沟通偏好。

价值：

- AI 不给泛泛答案，而给家庭专属答案。

#### 2.6.7 老人兴趣入口

为了让老人平时愿意用，可以加入兴趣功能：

- 听小说。
- 听评书。
- 听相声。
- 听戏曲。
- 看新闻。
- 看电视剧。
- 看短剧。
- 写节日祝福。
- 写朋友圈文案。
- 推荐菜谱。
- 陪聊。

判断：

- 这些功能不能作为第一版核心。
- 它们适合做留存入口。
- 防骗和截图陪跑才是更强付费点。

#### 2.6.8 快手/抖音短剧辅助

想法来源：

- 老人可能不想用 AI 助手，但喜欢刷快手、抖音、短剧。

可做功能：

- 识别短剧截图。
- 告诉老人当前页面怎么找合集。
- 判断评论区“加微信看全集”是否有风险。
- 找不到时转给子女帮忙。
- 记录老人喜欢的题材和安全内容源。

问题：

- 如果不能直接找到某部短剧下一集，价值会明显下降。
- 快手/抖音内容分散，接口和版权复杂。

结论：

- 短剧功能适合作为老人助手的娱乐入口，不适合单独做核心，除非能稳定直达内容。

#### 2.6.9 网购、退款、直播购物辅助

老人常见问题：

- 买错东西不会退。
- 找不到订单。
- 不懂“退款”和“退货退款”区别。
- 商家让加微信退款，容易被骗。
- 直播间买完找不到售后。
- 平台客服、商家客服、机器人客服混在一起看不懂。

可做：

- 截图订单页或聊天页。
- AI 指导下一步点哪里。
- 涉及退款、转账、加微信时提醒子女。

#### 2.6.10 微信、弹窗、权限、账号安全

可做场景：

- 手机弹窗能不能点。
- App 权限能不能开。
- 微信/支付宝/银行卡提示看不懂。
- 被要求下载远程控制软件。
- 被诱导发验证码。

### 2.7 和豆包、Kimi、元宝的区别

通用 AI：

- 是一个 App。
- 老人需要主动下载、打开和学习。
- 不知道家庭关系。
- 不会通知子女。
- 不知道父母的习惯、常用医院、风险规则。

本项目：

- 嵌入微信/类微信入口。
- 有子女端。
- 有家庭档案。
- 有风险规则。
- 高风险时通知子女。
- 能做截图陪跑。
- 能结合家庭知识库回答。

一句话：

> 豆包是回答问题的 AI；这个产品是连接老人、子女和现实生活流程的家庭助手。

### 2.8 最大风险

老人使用意愿不确定。

真实反馈：

- 有些老人觉得完全用不上。
- 有问题直接问别人。
- 不会网购就去实体店。
- 怕被骗就不理。
- 每天刷快手看短剧就行。

结论：

- 不能指望老人主动爱上 AI 助手。
- 更现实的是子女侧驱动，老人侧低感使用。
- 产品不要做成监控老人，而要做成“低门槛求助入口”。

隐私边界：

- 不能全面监控老人每天干了什么。
- 不能自动知道老人刷了什么快手。
- 不能自动知道老人有没有转账。
- 不能自动读取购物订单。
- 不能自动监控微信聊天。
- 只能服务老人主动触发的求助和明确授权的提醒。

### 2.9 第一版 MVP

第一版只做：

老人端：

- 模拟微信聊天界面，或先用企业微信/微信客服/公众号服务号。
- 支持文字、语音、图片、截图。
- 可疑短信/链接/验证码/转账/退款/投资内容判断。
- 手机截图一步一步指导。
- 高风险时明确提醒“先不要点、不要转钱、不要输验证码”。

子女端：

- 创建父母档案。
- 填写城市、常见风险、紧急联系人。
- 设置高风险提醒规则。
- 接收高风险通知。
- 查看重要摘要。

暂时不做：

- 完整陪聊。
- 复杂家庭知识库。
- 复杂用药管理。
- 听小说/电视剧推荐。
- 完整远程介入。
- 机构合作系统。

### 2.10 验证问题

必须验证：

1. 老人是否愿意把可疑内容和截图发给这个助手。
2. 企业微信/微信客服/公众号服务号形态是否足够像好友。
3. 子女是否愿意为“父母少被骗、少折腾、有风险能提醒我”付费。
4. 截图陪跑和防诈骗提醒是否真的降低子女负担。
5. 隐私授权和知情同意是否能设计清楚。

### 2.11 评分

| 维度 | 分数 | 判断 |
| --- | ---: | --- |
| 需求真实性 | 8.5 | 老人防骗、手机不会用、子女远程照护是真痛点。 |
| 市场趋势 | 8 | 老龄化、独居老人、子女异地支持它。 |
| 付费可能 | 6.5 | 子女可能付费，但要证明真的省心。 |
| MVP 可做性 | 7 | 可先用网页 + 微信客服/企微 + AI 验证。 |
| 微信生态可控性 | 4.5 | 第三方很难做成真正微信好友。 |
| 技术壁垒 | 5.5 | 早期不高，壁垒在场景、信任和数据。 |
| 合规/隐私风险 | 5 | 聊天记录、老人授权、防骗和医疗提醒敏感。 |
| 长期空间 | 8 | 如果跑通家庭照护入口，后续空间大。 |

综合：

- 作为马上创业融资项目：约 6/10。
- 作为大二学生 MVP、比赛、作品集、早期探索：约 8.5/10。
- 综合：约 7.2/10。

### 2.12 更可能赚钱的老人方向

优先级：

1. 父母反诈 + 截图陪跑 + 子女提醒。
2. 反诈与数字安全助手。
3. 慢病随访 + 用药/复诊管理。
4. 陪诊/护理/家政调度 SaaS。
5. 智能硬件家属端和异常提醒平台。
6. 社区养老数字化系统。
7. 养老机构管理系统。
8. 老年内容娱乐/陪伴。

最容易亏钱：

- 直接向老人卖独立 App。

更可能挣钱：

- 帮子女、机构、医院、保险、社区降低照护成本和风险。

## 3. 项目二：AI 运动动作纠正教练

### 3.1 一句话定位

上传一段运动视频，AI 分析姿势问题，给出风险点和纠正建议。第一版可从深蹲或篮球投篮切入。

### 3.2 原始想法：篮球投篮教练

功能：

- 用户上传投篮视频。
- 系统分析投篮姿势。
- 给出纠正建议。
- 匹配最相近的球星投篮风格。

卖点：

> 上传你的投篮视频，看看你最像哪位球星，并找出影响命中率的 3 个动作问题。

### 3.3 篮球方向功能

可分析：

- 出手点。
- 肘部角度。
- 手腕跟随动作。
- 下肢发力。
- 身体是否歪。
- 是否前倾。
- 起跳和落地。
- 最像哪个球星。
- 训练建议。

优点：

- 年轻用户愿意试。
- 展示效果强。
- “像哪个球星”有传播性。
- 适合小红书、抖音、B 站、篮球社团测试。

问题：

- 动作快，关键帧难抓。
- 投篮标准个体差异大。
- 球星匹配不准容易像玩具。
- 用户可能只试一次。

### 3.4 拓展方向：所有运动动作纠正

可能覆盖：

- 深蹲。
- 俯卧撑。
- 平板支撑。
- 弓步蹲。
- 跑步。
- 篮球投篮。
- 硬拉。
- 卧推。
- 引体向上。
- 瑜伽/普拉提。
- 羽毛球/网球挥拍。

原则：

> 越泛越难，越具体越容易起步。

不要第一版做“所有运动”。先做一个动作。

### 3.5 最容易做的动作：深蹲

深蹲最适合 MVP。

原因：

- 动作慢。
- 容易识别。
- 标准相对明确。
- 用户容易理解。
- Demo 展示效果好。

可分析：

- 膝盖是否内扣。
- 膝盖是否晃动。
- 背部是否过度弯曲。
- 下蹲深度。
- 脚跟是否离地。
- 身体是否过度前倾。
- 左右是否不对称。

推荐 MVP：

> AI 深蹲教练：上传一段侧面或正面深蹲视频，检测膝盖内扣、背部角度、下蹲深度、重心偏移，并给出纠正建议。

### 3.6 跑步姿态分析

跑步难点：

- 动作快。
- 人在移动。
- 关键帧难抓。
- 拍摄标准化难。

MVP 拍摄要求：

- 手机横屏。
- 固定机位。
- 距离跑者 4 到 6 米。
- 镜头高度在腰部到胸口。
- 跑者从画面左侧跑到右侧。
- 拍 5 到 8 秒。
- 全身入镜。
- 光线充足。

可分析：

- 身体前倾。
- 步幅。
- 触地点是否太靠前。
- 膝盖抬升。
- 后摆腿。
- 摆臂角度。
- 步频粗略估计。
- 垂直弹跳。

不要第一版承诺：

- 精准受伤风险诊断。
- 精确触地力。
- 足内翻/外翻精确判断。

### 3.7 MVP 优先级

1. 深蹲动作分析。
2. 篮球投篮娱乐化分析。
3. 跑步侧面姿态分析。

### 3.8 风险

- 专业性不足会损害信任。
- 拍摄角度影响结果。
- 运动建议可能涉及受伤风险。
- 竞品存在。
- 如果泛化太早，会失控。

### 3.9 适合场景

- 作品集。
- 大学生比赛。
- AI + 计算机视觉项目展示。
- 体育社团测试。
- 健身房/教练辅助工具原型。

## 4. 项目三：微信聊天记录/截图转纪念册、文档、证据包

### 4.1 一句话定位

上传几张微信聊天截图，AI 自动整理成一本可保存的 PDF 纪念册，也可以扩展成聊天证据包或工作文档。

### 4.2 原始需求

你自己有需求：

- 想把和某个人的微信聊天记录全部导出。
- 想转成某种东西保存起来。

### 4.3 现实边界

微信小程序不能直接读取用户和某个人的全部聊天记录。

可行导入方式：

1. 截图上传。
2. 长截图上传。
3. 复制粘贴。
4. 录屏滑动聊天。
5. PC 本地工具解析微信数据。

### 4.4 导入方式对比

截图上传：

- 最容易做。
- 用户麻烦。
- 适合精选聊天、纪念日、重要对话、证据整理。

长截图：

- 比普通截图方便。
- 需要自动切分、OCR、去重、排序。

复制粘贴：

- 实现最快。
- 但格式、图片、时间、表情容易丢。

录屏导入：

- 用户滑动聊天并录屏。
- 系统抽帧、去重、OCR。
- 体验比大量截图好，但技术更复杂。

PC 本地工具：

- 最接近完整导出。
- 技术和隐私难度最高。
- 不适合第一版。

### 4.5 可生成内容

纪念册版：

- 封面。
- 两个人名字。
- 时间范围。
- 第一句聊天。

## 5. 项目四：AI C 盘解释器/电脑空间与桌面整理助手

### 5.1 一句话定位

扫描电脑 C 盘、桌面和下载文件夹，用图和人话告诉用户空间被谁占了、每个文件夹是干什么的、哪些能清理、哪些不要动。

### 5.2 原始需求

你自己有需求：

- C 盘满了。
- 不敢乱删。
- 希望一眼看懂每个文件/文件夹是干什么的。
- 希望知道应该怎么处理。

核心痛点：

> 不是用户不知道 C 盘满了，而是不知道哪些东西能删、删了会不会出事。

### 5.3 产品定位

不要叫：

- AI 清理大师。
- 电脑优化大师。
- 一键清理神器。

更好的名称：

- AI C 盘解释器。
- 电脑空间看懂助手。
- C 盘看懂助手。
- AI 文件管家。
- DiskLens AI。

核心差异：

> 不是一键乱删，而是先解释清楚，让用户敢处理。

### 5.4 核心功能

#### 5.4.1 C 盘空间解释

功能：

- 扫描大文件夹。
- 展示 Top 占用目录。
- AI 解释来源和用途。
- 标记安全等级。
- 推荐处理方式。

示例：

- 微信文件：聊天图片、视频和文件，建议从微信设置里清理，不建议直接删数据库。
- Downloads：安装包、压缩包、视频，可按时间和类型整理。
- `.gradle`：Java/Android 构建缓存，可清理旧版本缓存。

#### 5.4.2 可视化空间图

可视化方式：

- Treemap 矩形树图。
- 树状目录图。
- 软件关系图。
- 安全等级颜色。

颜色：

- 绿色：基本可清理。
- 黄色：建议确认。
- 红色：不要手动删除。
- 蓝色：用户文件，可移动到其他盘。
- 灰色：未知，需要谨慎。

#### 5.4.3 桌面/下载文件夹整理推荐

功能：

- 扫描桌面和下载文件夹。
- 按文件类型、来源、时间、项目归类。
- AI 推荐文件夹结构。
- 找出重复文件、过期安装包、大文件。
- 建议移动到 D 盘、归档或删除。

推荐分类：

- 学习资料。
- 项目代码。
- 截图。
- 安装包。
- 压缩包。
- 待处理。
- 可删除。

#### 5.4.4 开发者专项清理

这个方向很适合软工学生。

识别对象：

- `node_modules`
- `.pnpm-store`
- `.npm`
- `.cache`
- `pip cache`
- `conda`
- `.gradle`
- `.m2`
- Docker images
- WSL distro
- VS Code 扩展缓存
- Android Studio SDK
- Unity/Unreal 缓存

差异化：

- 普通清理工具未必懂开发环境缓存。
- 可以面向学生、程序员、开发者。

### 5.5 安全边界

必须坚持：

- 默认只读扫描。
- 第一版不删除。
- 不自动移动文件。
- 不上传文件内容。
- AI 只解释路径、文件类型、大小、时间等元信息。
- 删除/移动必须用户确认。
- 高风险目录禁止一键删除。
- 删除优先放回收站。
- 支持撤销移动。

### 5.6 市场情况

已有产品：

- WinDirStat。
- SpaceSniffer。
- TreeSize。
- WizTree。
- Windows 存储感知。
- CCleaner。
- AiCleanDisk Pro。
- Nerion。
- ComputerDoc。
- WinPolish。
- Sortio。
- Filect。
- Filenest。
- AI Cubby。
- Foldora。

结论：

- 不是没人做。
- 但中文 Windows 用户、本土软件路径、开发者缓存、解释优先、安全分级这一块还有切口。

### 5.7 第一版 MVP

只做 Windows 本地只读扫描报告：

1. 扫描指定目录，如 C 盘、桌面、下载文件夹。
2. 展示 Top 50 占空间目录。
3. 用 Treemap 展示空间分布。
4. 给每个目录解释：
   - 这是什么。
   - 谁产生的。
   - 能不能删。
   - 推荐怎么处理。
5. 给桌面/下载文件夹生成整理建议。
6. 不执行删除，只导出整理报告。

一句话：

> 扫描你的 C 盘和桌面，生成一份电脑空间体检报告。

### 5.8 付费方向

可能模式：

- 免费基础扫描。
- 付费深度解释报告。
- 付费开发者专项清理。
- 付费桌面/下载文件夹整理方案。
- 一次性买断 29 到 99 元。
- 后续支持 Windows + macOS。

### 5.9 当前判断

优点：

- 你自己有强需求。
- 普通 Windows 用户痛点明确。
- 不依赖微信生态。
- 技术可控。
- 展示效果强。
- 适合作品集和比赛。

风险：

- 竞品多。
- 用户对清理软件不信任。
- 误删风险严重。
- 文件隐私敏感。
- AI 解释不能胡说。

综合：

> 这是目前综合排位很高的方向。关键不要做“AI 清理软件”，而要做“AI 电脑空间解释器”。

## 6. 项目五：可疑内容/AI 深度伪造风险判断助手

### 6.1 一句话定位

不做万能 AI 鉴定器，而做普通人可疑内容风险判断助手：用户上传可疑图片、语音、视频、聊天记录，系统判断是否存在诈骗或伪造风险，并给出核验步骤。

### 6.2 原始动机

AI 图片、视频、语音越来越真实，普通人很难判断真假。

风险场景：

- 冒充亲人。
- 冒充领导。
- 假新闻。
- 诈骗语音。
- 伪造证据。
- 投资骗局。
- 老人收到可疑内容。

### 6.3 不建议做的版本

不建议做：

> 上传图片/视频，判断是不是 AI 生成。

原因：

- 技术对抗强。
- 模型判断可能不稳定。
- 很难长期保证准确。
- 竞品压力高。

### 6.4 更好的切口

做具体场景里的风险判断：

- 这段语音是不是可能冒充亲人？
- 这个截图是否有诈骗风险？
- 这个视频里的要求是否可疑？
- 这个链接/二维码/聊天内容能不能信？

结合老人助手：

- 老人收到可疑图片、语音、视频、聊天记录时，系统不说“100% 假”，而是判断风险，并告诉下一步怎么核验。

| 付费可能 | 7 |
| MVP 可做性 | 7 |
| 技术壁垒 | 6 |
| 长期空间 | 8.5 |
| 竞争压力 | 8.5 |
| 综合 | 8.1 |

### 6.6 当前判断

- 方向很重要。
- 不适合第一优先级单独做。
- 更适合作为老人反诈助手或普通人安全助手的一部分。
